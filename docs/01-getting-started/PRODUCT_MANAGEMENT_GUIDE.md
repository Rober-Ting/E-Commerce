# 商品管理系统使用指南

## 📋 概述

本指南帮助你快速上手使用改进后的商品管理系统前端 Demo (`frontend_products_demo.html`)。

---

## 🚀 快速开始

### 步骤 1: 初始化测试账户

如果还没有创建测试账户，请先运行：

```powershell
.\init_users.ps1
```

输入 `y` 创建测试账户。

### 步骤 2: 启动服务

**终端 1 - 后端**:
```powershell
.\start_backend.ps1
```

**终端 2 - 前端**:
```powershell
.\start_products_demo.ps1
```

或使用：
```powershell
python -m http.server 8080
```

### 步骤 3: 访问系统

浏览器访问：`http://localhost:8080/frontend_products_demo.html`

## 🛍️ 创建商品

### 方法 1: 快速填充示例数据（推荐新手）

1. 使用 Vendor 账户登录
2. 点击右上角「➕ 创建商品」按钮
3. 在弹出的表单中点击「⚡ 快速填充示例数据」
4. 系统会自动填入一个完整的商品示例
5. 查看并修改（如需要）
6. 点击「💾 保存商品」

**优点**：
- ✅ 自动填入所有必填项
- ✅ 包含真实的图片 URL（来自 Unsplash）
- ✅ 格式正确的描述和标签
- ✅ 可以立即预览图片效果

### 方法 2: 手动填写完整商品信息

#### 📝 基本信息

**商品名称** *（必填）
- 清晰准确的商品标题
- 建议包含品牌、型号、规格
- 示例：`iPhone 15 Pro Max 256GB 原色钛金属`

**商品分类** *（必填）
- 可以从下拉列表选择现有分类
- 也可以输入新的分类名称
- 常用分类：手機、筆記型電腦、平板、電腦配件、耳機音響、智能手錶

**商品描述** *（必填）
- 详细描述商品的特点、功能、规格
- 建议至少 50 字
- 可以使用换行分段，提高可读性
- 示例：
```
Apple M3 Pro 芯片，具有 11 核中央处理器和 14 核图形处理器
18GB 统一内存
512GB 固态硬盘
14.2 英寸 Liquid Retina XDR 显示屏
三个 Thunderbolt 4 端口，HDMI 端口，SDXC 卡插槽
```

**标签关键词**（可选）
- 用逗号分隔
- 有助于搜索和推荐
- 示例：`Apple, MacBook, M3, 笔记本电脑, 专业工作站`

#### 💰 价格与库存

**销售价格** *（必填）
- 用户购买时的价格
- 支持小数点（例如：9999.99）
- 单位：元（人民币）

**库存数量** *（必填）
- 当前可售库存
- 整数
- 库存为 0 时建议将状态改为「缺货」

#### 📷 商品图片

**图片 URL**（推荐）
- 每行一个图片 URL
- 支持多张图片（建议 2-5 张）
- 输入后会自动预览
- 建议尺寸：800x800px

**如何获取图片 URL**：

1. **使用免费图床**：
   - [Imgur](https://imgur.com) - 上传图片后右键复制图片地址
   - [PostImages](https://postimages.org) - 支持直链
   - [ImgBB](https://imgbb.com) - 免费图床

2. **使用 Unsplash 免费图片**（测试用）：
   ```
   https://images.unsplash.com/photo-1517336714731-489689fd1ca8?w=800
   https://images.unsplash.com/photo-1496181133206-80ce9b88a853?w=800
   ```

3. **自建服务器**（生产环境）：
   - 将图片上传到自己的服务器
   - 使用 CDN 加速
   - 确保 HTTPS 安全访问

**图片要求**：
- ✅ 必须是 `http://` 或 `https://` 开头的完整 URL
- ✅ 支持格式：JPG、PNG、WEBP、GIF
- ✅ 建议使用 HTTPS（更安全）
- ✅ 图片应该清晰、美观、符合商品实际
- ❌ 不支持本地文件路径（如 `C:\images\product.jpg`）

#### ⚙️ 商品设置

**商品状态**
- ✅ **上架中** - 用户可以购买
- ❌ **已下架** - 暂不销售
- 📦 **缺货** - 暂时缺货

## 🔍 搜索与筛选

### 快速搜索
- 在顶部搜索框输入关键词
- 按 Enter 或点击「搜索」按钮
- 实时搜索商品名称、描述、标签

### 高级筛选

左侧筛选面板提供：

1. **搜索关键词** - 详细搜索
2. **商品分类** - 按分类筛选
3. **价格区间** - 设置最低价和最高价
4. **商品状态** - 筛选上架/下架/缺货
5. **排序方式** - 按价格、销量、评分、浏览量等排序
6. **排序方向** - 升序或降序

点击「应用筛选」生效，「重置」清除所有筛选条件。

---

## ✏️ 编辑商品

1. 在商品卡片上点击「编辑」按钮
2. 修改任意字段
3. 点击「💾 保存商品」

**注意**：编辑时会自动加载商品的现有信息，包括图片预览。

---

## 🗑️ 删除商品

1. 在商品卡片上点击「删除」按钮
2. 确认删除操作
3. 商品将被永久删除

**⚠️ 警告**：删除操作不可恢复！

---

## 👀 查看商品详情

点击「查看」或「查看详情」按钮，弹出包含完整信息的对话框：
- 商品名称、描述
- 价格、库存
- 分类、状态
- 销量、浏览量、评分
- 标签列表

页面顶部显示：

- 📁 **分类数** - 不同分类的数量

- 每页显示 12 个商品
- 底部显示当前页码和总页数
- 点击「← 上一页」或「下一页 →」切换
- 第一页时「上一页」按钮禁用
- 最后一页时「下一页」按钮禁用

## 💡 使用技巧

### 1. 快速测试流程

```
登录 Vendor 账户
  ↓
点击「创建商品」
  ↓
点击「快速填充示例数据」
  ↓
（可选）修改部分信息
  ↓
点击「保存商品」
  ↓
商品立即出现在列表中
```

### 2. 图片预览功能

- 输入图片 URL 后，会自动在表单中预览
- 如果图片加载失败，会显示"加载失败"提示
- 可以输入多个 URL 查看多图效果

### 3. 批量创建商品

- 多次点击「快速填充示例数据」会随机选择不同的示例
- 每次保存会创建一个新商品
- 适合快速填充测试数据

### 4. 搜索优化

- **标签**对搜索很重要，尽量填写准确的关键词
- 商品描述中的文字也会被搜索
- 使用分类筛选可以快速缩小范围

### 5. 图片选择建议

**测试环境**：
- 使用 Unsplash 免费图片
- 快速、方便、无需上传

**生产环境**：
- 使用自己的图片服务器
- 或使用云存储（如阿里云 OSS、腾讯云 COS）
- 确保图片稳定可访问

## 🎯 电商商品信息最佳实践

### 商品标题规范

```
✅ 好的标题：
- Apple iPhone 15 Pro Max 256GB 原色钛金属
- MacBook Pro 14" M3 Pro 芯片 18GB 512GB
- AirPods Pro (第二代) USB-C 无线降噪耳机

❌ 不好的标题：
- 手机
- 很好的笔记本
- 耳机便宜卖
```

### 商品描述规范

**包含以下内容**：
1. **核心特性**：最重要的卖点（3-5 点）
2. **技术规格**：详细参数
3. **适用场景**：谁适合使用
4. **包装清单**：包含什么配件

**格式建议**：
- 使用换行分段，提高可读性
- 用简短的列表展示特性
- 突出重点内容

### 图片规范

**数量**：2-5 张
1. **主图**：商品正面全景图
2. **细节图**：重要细节特写
3. **场景图**：使用场景展示
4. **包装图**：包装和配件

**质量要求**：
- 分辨率：至少 800x800px
- 格式：JPG 或 PNG
- 背景：纯色背景或真实场景
- 光线：明亮、清晰、无模糊

### 定价策略

- 参考市场价格
- 考虑成本和利润
- 整数定价（如 9999）或 .99 定价（如 9999.99）
- 保持价格竞争力

### 库存管理

- 实时更新库存数量
- 库存不足时及时补货
- 缺货时设置状态为「缺货」而不是删除商品

## ⚠️ 常见问题

### Q1: 为什么我看不到「创建商品」按钮？

**A**: 只有 **Vendor** 和 **Admin** 可以创建商品。
- 确认你使用的是 `vendor@test.com` 或 `admin@ecommerce.com` 登录
- Customer 账户只能查看商品，无法创建

---

### Q2: 图片 URL 填了但不显示？

**A**: 检查以下几点：
1. URL 是否以 `http://` 或 `https://` 开头
2. URL 是否可以在浏览器中直接访问
3. 图片服务器是否允许跨域访问
4. 检查网络连接

---

### Q3: 保存商品时提示错误？

**A**: 常见原因：
1. 必填项未填写（名称、描述、价格、库存、分类）
2. 价格或库存为负数
3. Token 过期，请重新登录
4. 后端服务未启动

---

### Q4: 如何上传本地图片？

**A**: 目前系统暂不支持直接上传文件。请：
1. 将图片上传到图床（如 Imgur）
2. 复制图片直链 URL
3. 粘贴到「商品图片 URL」字段

未来版本会支持文件直接上传功能。

---

### Q5: 删除了商品能恢复吗？

**A**: 不能。删除操作是永久性的。请谨慎操作。
